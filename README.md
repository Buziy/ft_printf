## Description
**ft_printf** is a project where we code our own version of the standard C library's `printf` function. 

Rather than relying on complex data structures or internal buffering, this implementation prioritizes a direct-to-output approach that is both lightweight and efficient.
## Supported Format Specifiers
This function handles the following conversions:
* `%c` Prints a single character.
* `%s` Prints a string.
* `%p` Prints a pointer address in hexadecimal format.
* `%d` Prints a decimal (base 10) number.
* `%i` Prints an integer in base 10.
* `%u` Prints an unsigned decimal (base 10) number.
* `%x` Prints a number in hexadecimal (base 16) lowercase format.
* `%X` Prints a number in hexadecimal (base 16) uppercase format.
* `%%` Prints a percent sign.

## Instructions

### Compilation
To compile the library, you just need to run:
```bash
make
```
This will generate a static library called libftprintf.a.

Other useful make commands:
```bash
make clean: Deletes the object (.o) files.

make fclean: Deletes the object files and the libftprintf.a library.

make re: Cleans everything and recompiles the project from scratch.
```
## Usage

To use this function in your own code, simply include the header file:
```c
#include "ft_printf.h"
```
And compile your project with the library:
```bash
cc your_file.c libftprintf.a
```
## Data Structure

The only specific data structure used in this project is `va_list` from the `<stdarg.h>` library, which is required to handle variadic functions. For printing numbers and hexadecimal values, the project uses **recursive functions**. This allows the numbers to be printed backward logically without using any extra external arrays, strings, or dynamically allocated memory (like `malloc`), keeping the code very clean and memory-efficient.
